Solution:

The classifiers can be represented using a confusion matrix which is a clear way to present the prediction results of a classifier.

Based on the data in this tables we can calculate the precision, recall, accuracy and the F-measure for each classifier.

Precision or Positive Predictive Value is the number of positive predictions divided by the total number of positive class values predicted. Precision can be thought of as a measure of a classifiers exactness.

\[Precision = \frac{TP}{TP+FP}\]

Recall is the number of positive predictions divided by the number of positive class values in the test data. It is also called Sensitivity or the True Positive Rate. Recall can be thought of as a measure of a classifiers completeness. A low recall indicates many False Negatives.

\[Recall = \frac{TP}{TP+FN}\]

Accuracy is the proportion of correct results that a classifier achieved.

\[Accuracy = \frac{TP+TN}{TP+TN+FP+FN}\]

The F1 score (F-score) is a measure of a test’s accuracy. The F1 score can be interpreted as a weighted average of the precision and recall, where an F1 score reaches its best value at 1 and worst at 0.

\[F1 = \frac{2*precision*recall}{precision+recall}\] The values for the precision, recall, accuracy and the F-measure are given in the following table.

Exercise 2

Plot the information about these classifiers on the “ROC-space”.

  1. Discuss the “goodness” of each classifier A, B, C, D using the scores from Task 1 and ROC space.

  2. Discuss how does the imbalance of different labeled examples potentially impact the goodness measures?

Solution:

  1. ROC curve, is a graphical plot that illustrates the performance of a binary classifier system. To draw a ROC curve, only the true positive rate (TPR) and false positive rate (FPR) are needed. The TPR defines how many correct positive results occur among all positive samples available during the test. FPR, on the other hand, defines how many incorrect positive results occur among all negative samples available during the test.

The best possible prediction method would yield a point in the upper left corner or coordinate (0,1) of the ROC space, representing 100% sensitivity (no false negatives) and 100% specificity (no false positives). The (0,1) point is also called a perfect classification. A random guess would give a point along a diagonal line (the so-called line of no-discrimination) from the left bottom to the top right corners.

A random guess would give a point along a diagonal line from the left bottom to the top right corners. Points above the diagonal represent good classification results (better than random), points below the line represent poor results (worse than random).

TPR and FPR can be calculated using:

\[TPR = \frac{TP}{TP+FN}\]

\[FPR = \frac{FP}{FP+TN}\]

# x calculated values for FPR, y calculated values for TPR for A,B,C,D
d <- data.frame(x = c(0.636363636, 0.272727273, 0.235294118, 0.411764706), y = c(0.692307692, 
    0.553846154, 0.666666667, 0.8), names = c("A", "B", "C", "D"))

ggplot(d, aes(x, y)) + geom_point() + geom_text(aes(label = names, hjust = 2)) + 
    xlab("FPR") + ylab("TPR") + ggtitle("ROC Space") + theme(plot.title = element_text(hjust = 0.5)) + 
    geom_abline(intercept = 0, slope = 1, linetype = 2, colour = "Red") + xlim(0, 
    1) + ylim(0, 1)

From the plot we can see that C is closest to the point (0,1). However from the metrics table in task 1 we can see that this classifier has very low precision and low F1 score. B is located closely to C and from task 1 we know that this is the best classifier. A had high values for the precision and the F1 score (compared to the other classifiers) but according to it’s location in the ROC space this classifier is close to being as good as random. D has the lowest precision and F1 score meaning that it is the worst classifier.

  1. Accuracy measures the ratio of correct predictions to the total number of cases evaluated. Although it seems obvious that the ratio of correct predictions to cases should be a key metric that is not the case. A predictive model may have high accuracy, but be useless. This happens when we have a big difference in the number of (true or false) predicted negative and predicted positive cases. Then the accuracy can be misleading. Therefore it may be better to avoid the accuracy metric in favor of other metrics such as precision and recall.

From the plots and the auc scores it is obvious that the models performance is really bad. The reason for that is that for some reason the scores are assigned in a way that we give higher score for the FALSE guesses instead of the TRUE guesses. The optimal ROC curve would have only 1’s first when we sort the data and then only 0’s in which case we would have 100% sensitivity (no false negatives) and 100% specificity (no false positives). We can make this the classifiers better by inverting its prediction (i.e. invert the order of returning 0 and 1, then the position of the points in the ROC space would be (1- fpr, 1- tpr)).

Based on the auc scores Classifier 2 has the largest area under the curve meaning it is the “best” classifier and Classifier 1 is the worst.

Type I error is the incorrect rejection of a true null hypothesis (a “false positive”, true 0 predicted 1), while a type II error is incorrectly retaining a false null hypothesis (a “false negative”, true 1 predicted 0). In other words, a type I error is detecting an effect that is not present, while a type II error is the failure to detect an effect that is present.

Exercise 5

Three different classifiers were presented individually in 3. Try to make an ensemble learner out of three. Try out two different methods of the four examples:

  1. take the simple sum of three scores

  2. scale the scores in order to weight more the “best” classifiers.

  3. convert scores to ranks and take the sum of three ranks

  4. use the sum of two best (or worst) ranks (out of three) as the new rank order.

Visualize on the same ROC curve plot with 3 individual plots. Which “ensemble” is your favorite?

Solution:

For this task I will use a) and b) to create the ensemble. First we need to calculate the sum and the weighted sum from the classifiers score. The classifiers who have higher auc values (calculated in task 3) are assigned with a higher number for the weights.

for (i in 1:nrow(X)) {
    row <- X[i, ]
    sum = (X[, 3] + X[, 4] + X[, 5])
    X$sum = sum
}

for (i in 1:nrow(X)) {
    row <- X[i, ]
    weight_sum = (1.4 * X[, 3] + 2 * X[, +4] + 1.7 * X[, 5])
    X$weight_sum = weight_sum
}

Then we need to sort the scores.

newdata1 <- X[order(sum), ]

sum_sort <- newdata1$sum
cls_sort1 <- newdata1$cls

newdata2 <- X[order(weight_sum), ]

wsum_sort <- newdata1$weight_sum
cls_sort2 <- newdata2$cls

Using the sorted scores we can plot the ROC curves and calculate the area under the curve.

pred1 = prediction(sum_sort, cls_sort1)
roc1 = performance(pred1, "tpr", "fpr")

pred2 = prediction(wsum_sort, cls_sort2)
roc2 = performance(pred2, "tpr", "fpr")

plot(roc1, lwd = 2, col = "red", main = "ROC Curves")
plot(roc2, lwd = 2, col = "blue", add = TRUE)
lines(x = c(0, 1), y = c(0, 1), col = "black", lwd = 1)

legend("topleft", legend = c("sum", "weighted sum"), col = c("red", "blue"), 
    lty = 1, cex = 0.6)

auc1 = performance(pred1, "auc")
auc1 = unlist(auc1@y.values)
# sum
auc1
## [1] 0.2543179
auc2 = performance(pred2, "auc")
auc2 = unlist(auc2@y.values)
# weighted sum
auc2
## [1] 0.386968

Based on the auc values we can conclude that the weighted sum ensemble gives the best results, compared to the previous three models and the sum ensemble.

    rank1 = 1:nrow(X)
    X$rank1 = rank1

X <- X[order(ROC2order), ]

    rank2 = 1:nrow(X)

X <- X[order(ROC3order), ]

for (i in 1:nrow(X)) {
    rank3 = 1:nrow(X)

# order all ranks

# c sum of all 3 ranks
for (i in 1:nrow(X)) {

    X$sum = sum
}

# d sum of 2 ranks

    X$sum2 = sum2
